Input Component

View Source

The dotbox-input component provides flexible input fields with various options for labels, icons, and validation.

Examples

Basic Usage

<dotbox-input label="Username" placeholder="Enter username"></dotbox-input> <dotbox-input label="Email" type="email" placeholder="Enter email"></dotbox-input> <dotbox-input label="Password" type="password" placeholder="Enter password"></dotbox-input>

Label Positions

<dotbox-input label="Label Above" labelPosition="above" placeholder="Default position"></dotbox-input> <dotbox-input label="Label Before" labelPosition="before" placeholder="Label on the left"></dotbox-input>

With Icons

<dotbox-input label="Username" icon="fa-user" placeholder="With user icon"></dotbox-input> <dotbox-input label="Email" icon="fa-envelope" placeholder="With email icon"></dotbox-input> <dotbox-input label="Password" type="password" icon="fa-lock" placeholder="With lock icon"></dotbox-input> <dotbox-input label="Search" icon="fa-search" iconPosition="after" placeholder="Icon on the right"></dotbox-input>

Input Types

<dotbox-input label="Text" type="text" placeholder="Text input"></dotbox-input> <dotbox-input label="Number" type="number" placeholder="Number input"></dotbox-input> <dotbox-input label="Email" type="email" placeholder="Email input"></dotbox-input> <dotbox-input label="Password" type="password" placeholder="Password input"></dotbox-input> <dotbox-input label="Date" type="date"></dotbox-input> <dotbox-input label="Search" type="search" placeholder="Search input"></dotbox-input>

Multiline Input

<dotbox-input label="Description" multiline="true" placeholder="Enter description" rows="4"></dotbox-input>

Disabled State

<dotbox-input label="Disabled Input" disabled="true" value="This input is disabled"></dotbox-input> <dotbox-input label="Disabled Textarea" multiline="true" disabled="true" value="This textarea is disabled"></dotbox-input>

Validation

<dotbox-input label="Required Field" required="true" placeholder="This field is required"></dotbox-input> <dotbox-input label="Email Validation" type="email" required="true" placeholder="Enter a valid email"></dotbox-input> <dotbox-input label="Min Length" minLength="8" placeholder="Minimum 8 characters" errorMessage="Please enter at least 8 characters"></dotbox-input> <dotbox-input label="Pattern" pattern="[A-Za-z]{3}" placeholder="Exactly 3 letters" errorMessage="Please enter exactly 3 letters"></dotbox-input>

Login Form Example

Login
<dotbox-input label="Username" icon="fa-user" required="true" placeholder="Enter your username"></dotbox-input> <dotbox-input label="Password" type="password" icon="fa-lock" required="true" placeholder="Enter your password"></dotbox-input> <dotbox-button variant="primary" style="width: 100%;">Login</dotbox-button>

Manual Validation

Set Valid Set Invalid
<dotbox-input id="my-input" label="Username" placeholder="Enter username"></dotbox-input> <script> // Set as valid document.getElementById('my-input').setValid(); // Set as invalid with custom message document.getElementById('my-input').setInvalid('This username is already taken'); </script>

API Reference

Properties

Name Type Default Description
label String "" Input label text
labelPosition String "above" Position of the label (before, above)
disabled Boolean false Whether the input is disabled
multiline Boolean false Whether the input should be a textarea
type String "text" Input type (text, number, password, email, tel, url, search, date)
icon String "" FontAwesome icon name (e.g., 'fa-user', 'fa-key')
iconPosition String "before" Position of the icon (before, after)
placeholder String "" Placeholder text
value String "" Input value
required Boolean false Whether the input is required
pattern String "" Validation pattern (regex)
errorMessage String "" Custom error message for validation
rows Number 3 Number of rows for textarea (only applies when multiline is true)
minLength Number null Minimum length of input
maxLength Number null Maximum length of input

Methods

Name Parameters Description
checkValidity() None Checks if the input is valid
focus() None Focuses the input element
reset() None Resets the input to its initial value
setValid() None Manually sets the input as valid
setInvalid(message) message (optional): String Manually sets the input as invalid with an optional custom error message

Events

Name Description
input Fired when the input value changes
change Fired when the input loses focus and the value has changed
focus Fired when the input gains focus
blur Fired when the input loses focus

CSS Parts

Name Description
container The container element
label The label element
input-wrapper The wrapper around the input
input The input element
icon The icon element
error The error message element
